# 코인 투자 감정 게임 (A안) — "패닉에 빠진 투자자를 진정시켜라"

> 대주제: **감정(Emotion)** · Upbit API 필수 · 띠리링 패턴 재활용
> 플랫폼: **웹 (데스크톱 기준)**
> 입력: **선택지 위주 + 학습 레이어**

---

## 0. 한 줄 정의

플레이어가 **투자 상담가**가 되어, 실제 과거 폭락·급등장에서
공포와 탐욕에 휩쓸리는 **AI 투자자**를 진정시킨다.

진정시키려면 **차트·뉴스·지표를 읽을 줄 알아야 한다.**
상담하는 동안 자연스럽게 **코인 보는 눈**을 배우고,
엔딩에서 **자기 자신의 투자 성향**을 마주한다.

---

## 1. 세 기둥

```
① 감정   — AI 투자자의 공포↔탐욕 게이지를 다룬다 (게임의 심장)
② 학습   — 진정시킬 근거를 차트·뉴스·지표에서 찾는다 (교육 가치)
③ 거울   — 남을 다룬 방식이 곧 내 성향을 드러낸다 (엔딩 반전)
```

미연시(캐릭터·연출) + 상담시뮬(게이지·분석) + 듀오링고(학습·즉각 피드백)이
이 세 기둥 위에서 하나로 묶인다. 미연시 톤과 분석 게이지는 **균형**.

---

## 2. 후킹 메시지

> **"-40%. 패닉에 빠진 투자자가 당신에게 전화를 걸어왔다."**
> *"당신은 이 사람을 살릴 수 있을까?"*

투자 안 해본 사람도 "남 말리는 건 할 수 있을 것 같은데?" 하고 진입.

---

## 3. 핵심 발상

기존 투자 게임은 "내 감정을 측정"한다. 이건 **"남의 감정을 다뤄야"** 한다.
- 플레이어는 한 발 떨어져 패닉을 **객관적으로** 본다
- 남에겐 "팔면 안 되지"라면서 정작 본인은 똑같이 패닉할 거라는 **아이러니**
- 그 간극이 곧 재미이자 교훈

---

## 4. 한 턴의 흐름 (학습 레이어 통합)

핵심: **선택지를 누르기 전에 정보를 읽도록** 한다.
상담이 "감"이 아니라 "근거"로 이뤄지고, 그게 곧 학습이 된다.

```
① 상황 발생
   AI 투자자가 패닉/흥분한 채 말을 건다
   "BTC가 -40%예요... 끝났어요 ㅠㅠ 지금이라도 팔까요?"
        ↓
② 정보 패널 공개  ★학습 구간★
   ├ 차트 (Upbit 실데이터, 현재 턴까지)
   ├ 뉴스 (해당 시점 실제 사건)
   ├ 공포탐욕지수
   └ [💡 도움말] — 개념을 모르면 탭해서 1~2줄 설명 확인
        ↓
③ 상담 선택지 (정보를 읽었다는 전제)
   A. 근거 기반 진정 ("지금 지수가 극단적 공포예요…")
   B. 동조 ("맞아요, 파는 게…")
   C. 무근거 위로 ("무조건 존버!")
        ↓
④ AI 감정 게이지 변동 + 반응  ★즉각 피드백★
   화면에 변동 수치 팝업: 공포 -15 ▼
        ↓
⑤ 게이지 임계치 돌파 → AI 매매 실행 → 손익 확정
        ↓
   다음 턴 (시간 경과 → 차트 다음 구간 공개)
```

**💡 도움말 시스템이 학습의 핵심.** 선택지에 나온 개념(공포탐욕지수,
지지선, 거래량, 데드캣 바운스 등)을 모르면 탭해서 짧은 설명을 보고 누른다.
→ 강제 학습이 아니라 **필요할 때 배우는** 자연스러운 구조.

## 5. 감정 게이지 (핵심 메커니즘)

```
         공포 ◀━━━━━━━━━●━━━━━━━━━▶ 탐욕
                      (중립=합리)
```

- 유저 선택지가 게이지를 좌우로 민다
- **공포 임계치 돌파** → AI 패닉셀 (바닥에서 던짐)
- **탐욕 임계치 돌파** → AI 추격매수/풀매수 (고점에서 물림)
- **중립 유지** → AI 합리적 판단, 자산 보존

> 게이지가 극단으로 가면 유저가 못 막고 **AI가 사고를 친다.**
> 이 "통제 불능의 순간"이 긴장감.

**중요:** 게이지·손익 계산은 **우리 코드가** 한다 (LLM은 대사만 생성).
→ 결과 일관성 확보 + 정량 측정 가능.

---

## 6. AI 투자자 페르소나

게임 시작 시 선택 또는 GPT 생성. 성향별 공략법이 다름:

| 캐릭터 | 약점 | 공략 핵심 | 표정 연출 |
|--------|------|-----------|-----------|
| 김불안 | 공포 | 안심시킬 근거 제시 | 떨림 → 안도 |
| 박욕심 | 탐욕 | 흥분 가라앉히기 | 눈 돌아감 → 차분 |
| 이초보 | 정보 휩쓸림 | 뉴스 거르는 법 코칭 | 혼란 → 이해 |
| 최존버 | 고집 | 때론 움직이게 | 무표정 → 흔들림 |

- 미연시처럼 감정 상태에 따라 표정/스탠딩 변화 (DALL·E 또는 에셋, 선택)
- 대사는 띠리링식 JSON 강제 출력으로 페르소나 일관성 유지

## 7. Upbit API 사용처

AI 투자자의 패닉/탐욕에 **"실제 근거"를 부여**하는 핵심 데이터.
AI는 추상적으로 무너지지 않는다 — **진짜 -40% 차트**를 보며 무너진다.

### 시나리오 (실제 사건)
| 시나리오 | 감정축 | 학습 포인트 |
|----------|--------|-------------|
| 21년 도지코인 광풍 | 탐욕 | 밈코인, FOMO, 일론 트윗 |
| 21년 코인 고점 | 탐욕→공포 | 천장 신호, 분할 매도 |
| 22년 거래소 파산 | 공포 | 패닉셀, 공포탐욕지수, 바닥 |
| 25년 하반기 | 혼합 | 최근장 응용 |

각 시나리오가 **하나의 "감정 + 개념 묶음"**으로 설계됨.

---

## 8. 엔딩 — 거울 반전 + 학습 리포트

게임 내내 (1) 선택 패턴 (2) 도움말 열람 (3) 게이지 안정 성과를 기록.

```
① 결과 — AI를 살렸는가, 같이 망했는가
   전체 차트 + AI 매매 시점 표시
   "실제 시장은 이랬습니다" 공개

② 거울 — 유저 본인의 성향
   "당신은 남에겐 '존버하라'고 했습니다.
    그런데 당신은 공포는 잘 다뤘지만,
    탐욕 앞에선 같이 흥분했습니다."

③ 학습 리포트 — 이번 판에서 마주친 개념
   ✓ 공포탐욕지수  ✓ 지지선  ✓ 데드캣 바운스
   "다음 시나리오에선 이걸 응용해보세요"
```

## 9. 정량 측정 (A안 약점 해결책)

| 측정 항목 | 방법 |
|-----------|------|
| 감정 제어력 | 유저 개입 후 게이지가 평균 몇 % 안정됐는가 |
| 성향 분석 | 선택지 비율 (근거기반/동조/무근거 위로) |
| 학습도 | 도움말 열람 수 + 근거기반 선택 비율 |
| 실력 성장 | 턴이 갈수록 근거기반 선택이 느는가 |

→ "왜 감정 게임이냐"엔 **게이지 안정도 수치**로 답.
→ "교육 가치는?"엔 **학습 리포트**로 답.

## 10. 기술 구조 (띠리링 재활용)

```
백엔드리스 — OpenAI API 직접 호출
매 턴 응답 JSON 강제 출력:
{
  "emotion_delta": -15,
  "current_emotion": "공포",
  "dialogue": "그 지수 얘기 들으니 좀 진정되네요...",
  "face": "relieved",
  "action": null            // 임계치 넘으면 "SELL_ALL" 등
}
이전 맥락은 핵심만 추출해 함께 전송 (띠리링 방식)

게이지·손익·차트 = 우리 코드 + Upbit 데이터로 계산
도움말 사전 = 미리 작성한 개념 설명 DB (LLM 호출 X, 빠르고 정확)
```

## 11. 화면 구성 (웹/데스크톱)

대화는 **3분할 레이아웃**. 순수 채팅이 아니라 대화 영역만 채팅 느낌.

- **왼쪽 (학습)** — 차트·공탐지수·뉴스 고정 패널 + 💡 도움말. 정보가 휘발 안 됨
- **가운데 (감정)** — AI 캐릭터 + 말풍선 대사 + 선택지
- **오른쪽 (분석)** — 감정 게이지 + 즉각 피드백 + AI 자산/손익

화면 목록:
- [ ] 시작 화면 (전화 오는 연출 + 후킹 카피)
- [ ] 캐릭터/시나리오 선택
- [ ] 메인 상담 화면 (3분할)
- [ ] AI 매매 실행 연출
- [ ] 엔딩 3단 (결과 / 거울 / 학습 리포트)

## 12. 구현 우선순위 (해커톤)

```
P0 (필수 동작)
  └ 1개 시나리오(22년 파산) + 1캐릭터 + 게이지 + 선택지 + 엔딩

P1 (차별화)
  └ 도움말 학습 시스템 + 거울 분석 + 정보 패널

P2 (여유되면)
  └ 표정 일러스트 + 시나리오 4개 풀 + 효과음/연출
```

P0만 돼도 "플레이 가능한 데모"가 나오게 설계.

## 13. 미결정 사항

- [ ] 도움말: LLM 생성 vs 사전 DB → **사전 DB 추천** (빠르고 정확)
- [ ] 캐릭터 일러스트: 직접 제작 vs DALL·E vs 무료 에셋
- [ ] 한 시나리오 턴 수 (8~12 사이)
- [ ] 기술 스택 (프론트 프레임워크, 차트 라이브러리)
