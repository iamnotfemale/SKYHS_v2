# 코인 투자 감정 게임 — "패닉에 빠진 투자자를 진정시켜라"
### 기획서 v2 (결정사항 확정판)

> 대주제: **감정(Emotion)** · Upbit API 필수 · 띠리링 패턴 재활용
> 플랫폼: **웹 (데스크톱)** · LLM: **Claude API**
> 입력: **조언 선택 + 근거 선택** · 턴 수: **10턴**

---

## 0. 한 줄 정의

플레이어가 **투자 상담가**가 되어, 실제 과거 폭락·급등장에서
공포와 탐욕에 휩쓸리는 **AI 투자자**를 설득한다.

유저는 조언에 **근거 자료**를 붙여 LLM의 **신뢰도(합리적 판단 확률)**를 기울이고,
LLM은 그 확률로 스스로 매매한다. 그리고 결과를 **복기하며 가르쳐준다.**
반복하면서 유저는 **차트와 시장 보는 법**을 배운다.

```
① 감정   — 패닉/탐욕에 빠진 LLM의 신뢰도를 설득으로 기울인다

```

```

        ↓

        ↓
⑤ 정합성 판정 → LLM 신뢰도 변동
   "김불안의 합리적 판단 확률: 35% → 68%"
        ↓

        ↓

   "공포지수가 바닥권이었는데 제가 흔들렸네요... 배웠어요"
        ↓
   다음 턴
```

## 4. 신뢰도 시스템 ★게임의 심장★

### 4-1. 핵심 수치 (확정)
```
시작 신뢰도        : 40%
신뢰도 범위        : 0% ~ 100%
신뢰도 누적 방식    : 게임 내내 누적 (매 턴 리셋 X)
                    → 유저의 상담이 쌓여 LLM이 점점 성장하거나 망가짐
```

### 4-2. 정합성 점수표 (조언 3 × 근거 4)
조언·근거가 둘 다 선택지이므로 **모든 조합에 점수를 미리 설계**한다.
LLM은 신뢰도 계산에서 **완전히 빠진다** (대사만 생성).

```
조언:  A.버티기   B.손절(동조)   C.추가매수
근거:  ㄱ.공탐지수  ㄴ.차트지지선  ㄷ.뉴스  ㄹ.커뮤니티분위기

점수 폭: -30% ~ +30%

설계 규칙
  · 강한 근거(공탐지수·차트지지선) + 방향 맞는 조언  →  +20~30%
  · 약한 근거(커뮤니티 분위기)                       →  ±5% (거의 무효)
  · 조언과 근거 방향 불일치                          →  -10~30%
```

예시 (22년 파산, BTC -40%, 바닥권):
```
A+ㄱ = +30%  (최적: 공포 바닥 + 버티기)
A+ㄴ = +25%  (좋음: 지지선 + 버티기)
B+ㄷ = -15%  (악수: 단기 악재로 손절 부추김)
C+ㄹ =  -5%  (도박: 패닉 분위기 보고 추매)
```

### 4-3. 확률 → 행동 곡선
```
신뢰도 75% 이상 : 합리적 선택 우세 (가끔 흔들림)
신뢰도 40~75%  : 반반, 변동성 큼
신뢰도 40% 미만 : 패닉/탐욕 폭주 우세

→ 100%는 없다. 좋은 근거를 줘도 가끔 LLM이 사고친다 = 긴장감
→ 행동 후 Upbit 실데이터로 손익 확정
```

### 4-4. 왜 강한가
- **심사 방어:** 신뢰도는 결정론적 테이블이라 "LLM 기분따라 변하냐" 공격 무효
- **안정성:** LLM은 대사만 → 일관성·토큰 절약
- **학습:** 맞는 조언+맞는 근거 짝짓기가 곧 시장 보는 법

---

## 5. 사후 복기 = 학습 엔진

유저가 가르치는 게 아니라 **LLM이 깨달으며 유저도 같이 배운다.**
- 좋은 선택: "공포지수를 봤어야 했는데, 당신 말이 맞았네요"
- 나쁜 선택: "또 바닥에 팔았네요... 다음엔 지표를 볼게요"
- 교육을 캐릭터에 위임 → 거부감 없음. 게임 목표의 핵심 장치.

### 6-1. P0 시나리오 (데모 필수)
| 시나리오 | 감정축 | 선정 이유 |
|----------|--------|-----------|
| **22년 거래소 파산** | 공포 | 패닉셀 유혹 최대, 공탐지수 극단값으로 학습 명확 |

### 6-2. P1 추가 시나리오
21년 도지 광풍(탐욕) / 21년 고점(탐욕→공포) / 25년 하반기(혼합)

### 6-3. P0 캐릭터 (데모 필수)
| 캐릭터 | 약점 | 파산장 궁합 |
|--------|------|-------------|
| **김불안** | 공포 | 폭락장에서 패닉 → 진정이 핵심, 시나리오와 완벽 |

P1: 박욕심(탐욕) / 이초보(정보휩쓸림) / 최존버(고집)

### 6-4. 근거 자료 4종 (고정)
```
ㄱ. 공포탐욕지수   (정량·강한 근거)
ㄴ. 차트 / 지지선  (정량·강한 근거)
ㄷ. 뉴스           (맥락·중간 근거)
ㄹ. 커뮤니티 분위기 (감정·약한 근거 — 휩쓸림의 함정)
```
> 커뮤니티를 "약한 근거"로 둔 게 핵심 교훈: 군중 감정에 기대면 신뢰도가 안 오른다.

## 7. Upbit API 사용처

AI 패닉/탐욕에 **실제 근거**를 부여하는 핵심 데이터.
- 시나리오 구간의 실제 캔들 → 차트 렌더링
- 근거 자료 'ㄴ.차트/지지선'의 실데이터
- LLM 행동 후 손익 계산
- ⚠️ **21·22년 데이터 조회 사전 검증 필수** (안 되면 시나리오 무너짐)

```
   ✓ 공포탐욕지수 읽는 법  ✓ 지지선  ✓ 군중심리의 함정
```

```
LLM        : Claude API (대사·복기 생성 전용)
             ※ 모델·단가는 착수 전 공식 문서 확인
             https://docs.claude.com/en/api/overview

차트       : lightweight-charts (트레이딩뷰 느낌, 캔들에 최적)
             또는 Recharts (간단하면)

백엔드     : 백엔드리스 (Claude API 직접 호출, 띠리링 방식)
```

### Claude API 응답 JSON (강제 출력)
```json
{
  "dialogue": "끝났어요... 다 팔까요?",
  "face": "panic",
  "reflection": "공포지수가 바닥권이었는데..."
}
```
신뢰도·확률·행동·손익 = 전부 우리 코드. LLM은 말만 생성.

### 데모 안정성
- 핵심 시연 시나리오는 **Claude 응답 캐싱** → 라이브 API 장애 차단
- JSON 파싱 실패 시 폴백 대사 준비

## 11. 화면 구성 (웹/데스크톱)

대화는 채팅 구조, 정보는 고정 패널. 순수 채팅 아님.
- **정보 영역** — 차트 / 뉴스 / 공탐지수 / 커뮤니티 (항상 보임)
- **채팅 영역** — LLM 캐릭터 + 대사 + 사후 복기
- **상담 입력** — 조언 선택 → 근거 선택 → 신뢰도 변동 피드백
- **상태 표시** — 신뢰도 %, LLM 자산/손익, 턴 카운터(N/10)

화면 목록:
- [ ] 시작 화면 (전화 연출 + 후킹)
- [ ] 시나리오/캐릭터 선택
- [ ] 메인 상담 화면
- [ ] LLM 매매 실행 연출
- [ ] 엔딩 3단 (결과/거울/리포트)
